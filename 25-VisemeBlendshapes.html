Viseme → Blendshapes
(ARKit 52)

ARKit provides 52 facial blendshapes covering every region of the face. For lip sync, a viseme class activates a specific sparse subset of these blendshapes at defined weights. This lookup-table approach avoids neural inference at render time and produces deterministic, artist-tunable results.

Sparse Matrix Properties

Most visemes activate only 2–5 blendshapes out of 52. The jaw group and mouth group account for ~80% of all activations. Eye/brow/cheek shapes are never activated by phoneme-driven visemes (they are controlled by expression channels — separate from lip sync).

03

Blendshape Interpolation at Boundaries

At phoneme boundaries the blendshape vector transitions from one viseme's weights to the next. We interpolate each blendshape independently using the half-cosine window from App 24. The result is a per-frame blendshape array ready for renderer consumption.

Per-Blendshape Interpolation Preview
Select two visemes and see how each blendshape weight transitions between them over the blend window.
Output: Per-frame blendshape array
The output of the viseme-to-blendshape stage is a NumPy array of shape (N_frames × 52), where each row is a normalised float weight vector. This array is consumed by the renderer (Blender/UE5/ARKit-compatible character) to drive facial animation. App 26 (Temporal Smoothing) applies a final filter pass before rendering.
