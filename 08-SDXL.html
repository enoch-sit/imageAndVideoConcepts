01

Dual Text Encoders

SD 1.5 used a single CLIP-L encoder producing 768-dim embeddings. SDXL uses two encoders in parallel and concatenates their outputs — tripling the conditioning dimensionality.

CLIP-L (OpenAI)
Same encoder as SD 1.5
77 tokens · 768-dim per token
Penultimate layer features
Best at short, concrete prompts
OpenCLIP-G (LAION)
Larger model trained on LAION-5B
77 tokens · 1280-dim per token
Last layer features
Better at complex, long prompts

Concatenated Embeddings

Both encoders receive the same prompt text. Their token embeddings are concatenated along the feature dimension, giving the U-Net a 2048-dim representation per token position (768 + 1280). A separate pooled embedding from OpenCLIP-G is also added to the time embedding for global style conditioning.

Conditioning vector construction c = concat(CLIP_L(prompt), OpenCLIP_G(prompt)) [77 × 2048]
c_pool = pool(OpenCLIP_G(prompt)) → added to time embedding

Why two encoders?
CLIP-L is strong at object–attribute binding ("a red car with a blue door"). OpenCLIP-G captures style and composition better. Together they reduce "semantic drop-out" — where a single encoder ignores parts of a complex prompt. SDXL also gets a larger effective context (the two encoders independently process 77 tokens each).

Base + Refiner Pipeline

SDXL includes two separate U-Nets: a base model that generates the overall composition, and a refiner model that sharpens fine details in the last denoising steps.

What Each Model Does

  • Base U-Net (2.6B params): handles the high-noise regime (t = T to t = 0.2·T). It places the large compositional elements — subjects, backgrounds, general colour palette.
  • Refiner U-Net (2.3B params): handles the low-noise regime (t = 0.2·T to t = 0). It adds fine-grained details: texture, hair, eyes, skin pores. The refiner was trained specifically on the LAION-aesthetic high-quality subset.
Optional refiner
The refiner is optional — the base model produces a valid output on its own. In our A16 lab environment we run base only to save VRAM. The refiner requires a second 2.3B U-Net loaded simultaneously or sequentially.
03

Architectural Upgrades vs SD 1.5

SDXL is not simply a larger SD 1.5. Multiple architectural changes beyond the dual encoders and refiner separate the two.

Size Conditioning

SDXL conditions the U-Net on the original image resolution (h_orig, w_orig) and the current crop coordinates, in addition to the timestep. This teaches the model to synthesise images at varying aspect ratios and resolutions without artificially "seeing" only cropped 512-square patches during training.

Size conditioning vector (added to time embedding) c_size = [h_orig, w_orig, crop_top, crop_left, h_tgt, w_tgt]

Transformer Blocks in All Resolutions

SD 1.5 uses attention blocks only in the lowest (8×) resolution U-Net stage. SDXL adds transformer attention blocks at the 16× resolution stage as well, giving the model richer global structure modelling and substantially increasing parameter count.
