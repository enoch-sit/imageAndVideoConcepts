Whisper — Speech Recognition
& Timestamp Extraction

OpenAI Whisper (2022) is a large-scale, weakly-supervised ASR model trained on 680 000 hours of labelled audio. For the lip-sync pipeline, we use its word-level timestamp mode (whisper_timestamped / faster-whisper) to feed phoneme-aligned text into MFA.

01

Mel Spectrogram Input

Whisper standardises audio to 16 kHz mono and computes an 80-bin log-mel spectrogram. A fixed 30-second chunk window is used: short clips are zero-padded, long recordings are processed in overlapping 30s windows. This gives a 3 000-frame ×80-bin input tensor.

STFT Parameters

Whisper uses a 25 ms Hann window with 10 ms hop length. At 16 kHz this gives 400-sample windows with 160-sample hops. The 30-second chunk contains exactly 3 000 frames (30 × 100 frames/s). Each frame is a 201-point FFT, mapped to 80 mel bins via a mel filterbank.

Key STFT values sample rate : 16 000 Hz
n_fft : 400 (25 ms)
hop_length : 160 (10 ms)
n_mels : 80
chunk_length : 30 s → T = 3 000 frames
input shape : (80, 3000)
02

Encoder–Decoder Transformer

The encoder processes the fixed 3 000-frame mel spectrogram; the decoder auto-regressively generates text tokens (and optionally timestamp tokens). Whisper's decoder is the unique piece — it uses special <|timestamp|> tokens interleaved with word tokens to output timing information.

Encoder: 2-layer Conv stem + Transformer

Two 1-D convolution layers with GELU reduce the 3 000 time steps to 1 500, then a N-layer Transformer adds positional embeddings and applies self-attention. The encoder output is a 1 500×d_model sequence of contextualised audio features. Whisper medium uses 24 layers, d_model=1024.

Encoder sizes (Whisper model family) tiny : 4L ×384d — 39M params
base : 6L ×512d — 74M params
small : 12L×768d — 244M params
medium: 24L×1024d — 769M params
large : 32L×1280d — 1550M params

Decoder: Auto-regressive with timestamp tokens

The decoder uses cross-attention over encoder outputs. In standard ASR mode it outputs word tokens. In timestamp mode, the decoder alternates between <|t_start|> <word> <|t_end|> triplets. Timestamps are quantised to 20 ms bins (0–30 s, 1 500 bins). This is the foundation for word-level timing.

03

Word Timestamps → MFA Input

Whisper's native timestamps are word-level. For phoneme-level alignment we use whisper_timestamped or faster-whisper, which apply Dynamic Time Warping (DTW) on the cross-attention weights to refine sub-word timing. The output feeds directly into MFA (App 23).

DTW Timestamp Refinement

Cross-attention weights in each decoder layer form a T_dec × 1500 matrix linking output tokens to encoder frames. DTW finds the optimal monotonic path through this matrix, giving a fine-grained token→frame alignment. Converted from frames (10ms/frame) to seconds.

DTW alignment A[i,j] = cross_attn_weight[token_i, frame_j]
path = DTW(A) — monotonic alignment path
t_word = path[token_start] × 0.01 — seconds

Output Format for MFA

Whisper outputs a JSON list of word dicts with keys: word, start, end, probability. We convert this to a plain text file (words on newlines) and a corresponding audio file segment — the two inputs MFA requires for forced phoneme alignment.

Whisper → MFA hand-off Whisper output: [{"word":"Hello","start":0.14,"end":0.54},...]
MFA input file: hello world this is a test
MFA audio: same 16 kHz mono WAV segment
MFA output: phoneme-level TextGrid & JSON
Note on Whisper vs DeepSpeech2
Whisper is used here purely to get word-level timestamps for MFA. DeepSpeech2 (used by SadTalker) is a CTC-based model optimised for 29-dimensional phoneme probability output at each frame — ideal for direct audio-to-motion regression. Whisper is much larger but produces more reliable transcriptions for diverse speakers and accents.
