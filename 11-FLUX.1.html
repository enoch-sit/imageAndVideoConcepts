Rectified Flow — Straight Trajectories

Traditional diffusion models follow curved stochastic paths from noise to data. Rectified flow trains the model to follow the straightest possible interpolation between noise and image distributions — enabling fewer steps with no quality loss.

The Optimal Transport Interpolation

Rectified flow uses a simple linear interpolation path between a noise sample z and a data sample x₀:

Forward interpolation (Rectified Flow) x_t = (1−t)·x₀ + t·z where z ~ N(0,I), t ∈ [0,1]

The learned velocity field v_θ(x_t, t) ≈ z − x₀ (target velocity is constant)

Because the target velocity is constant along the path, numerical ODE solvers need fewer function evaluations. DDPM's curved paths require corrective steps at each point; tight linear paths can be solved in 1–4 steps.

Flow Matching vs Score Matching
DDPM/DDIM implicitly learn the score ∇ log p(x_t) — the gradient of the log probability. Rectified flow / flow matching learns the velocity v(x_t, t) = dx/dt of a deterministic ODE. Both are equivalent under certain conditions but flow matching gives cleaner training signal and straighter paths.

MMDiT — Multimodal Diffusion Transformer

FLUX.1 replaces the U-Net with a Transformer that processes image tokens and text tokens jointly in every layer, using separate weights for each modality but shared attention.

Interactive: MMDiT Token Flow
Click a block to see what happens to image tokens and text tokens inside an MMDiT layer.

Separate Weights, Shared Attention

In SD 1.5's U-Net, text cross-attention is applied after image self-attention — text is secondary. In MMDiT, both modalities run through the same attention matrix simultaneously. The Q/K/V projections use different weight matrices for image tokens (W_img) and text tokens (W_txt), but they are concatenated before the softmax, letting image and text tokens attend to each other directly.

MMDiT attention (single head) Q = [W_img_Q · x_img ; W_txt_Q · x_txt] (concat)
K = [W_img_K · x_img ; W_txt_K · x_txt]
V = [W_img_V · x_img ; W_txt_V · x_txt]
Attn = softmax(Q·Kᵀ/√d) · V (all tokens attend all)

Guidance Distillation

FLUX.1-schnell and FLUX.1-dev use guidance distillation to eliminate the need for separate conditional/unconditional forward passes (as in CFG). Instead, a guidance scale embedding g is passed directly to the model during training. At inference, a single forward pass with the desired g produces CFG-equivalent output, halving compute cost.

T5-XXL text encoder
FLUX.1 adds T5-XXL (4.7B parameters) alongside CLIP-L for text encoding. T5 is an encoder-decoder language model pre-trained on text-to-text tasks. Its deep language understanding dramatically improves adherence to long, complex prompts — especially counting, spatial relationships and rare concepts. Text tokens from T5 are passed directly to the MMDiT attention layers.

Why FLUX Matters for This Course

  • FLUX.1 sets the state-of-the-art baseline for image generation quality at the time of this course
  • Its trajectory framework (rectified flow) is inherited by video models including CogVideoX
  • Guidance distillation is the same technique used in SANA-Sprint and AnimateDiff-Lightning
  • The MMDiT architecture replaces U-Net in the newest stable video generation systems
