Forced Aligner

MFA (McAuliffe et al., 2017) performs forced phoneme alignment: given a transcript and its corresponding audio, it outputs precise start/end times for every phoneme. This produces the phoneme sequence with millisecond timing needed for phoneme-to-viseme mapping.

HMM-GMM Acoustic Model

MFA uses a Kaldi-based HMM-GMM acoustic model. Each phoneme is modelled as a 3-state HMM (beginning, middle, end) where each state emits frames according to a Gaussian Mixture Model over MFCC features. Viterbi decoding finds the most probable state sequence, producing phoneme-level boundaries.

MFCC Feature Extraction

MFA computes 13 MFCCs + delta + delta-delta (39 features) at 10ms frame intervals. The acoustic model was trained on large English corpora (LibriSpeech + Common Voice). For each input speaker, MFA can optionally adapt the model with a brief adaptation pass.

MFA feature pipeline audio (16kHz) → 25ms Hamming STFT → 40 mel bins
→ 13 MFCCs (DCT of log-mel) + Δ + ΔΔ = 39D
→ CMVN normalisation per utterance
→ 3-state HMM per phoneme × GMM emission model
02

TextGrid Output Format

MFA writes a Praat TextGrid file containing two tiers: a word tier (word-level intervals) and a phoneme tier (phoneme-level intervals). Each interval has start time, end time, and label. We convert this to JSON for downstream processing.

Phoneme Set → Lip Sync Pipeline

MFA uses the CMU Pronouncing Dictionary phoneme set (ARPAbet): 39 phonemes covering all English sounds. These map 1-to-many to 15 viseme classes in App 24. Click a phoneme group to preview its viseme class.

How Whisper + MFA work together
Whisper provides the transcript (Whisper transcribes better than MFA's built-in lexicon for spontaneous speech). MFA provides phoneme-level timing (Whisper's timestamps are word-level only). Together they give the phoneme sequence with timecodes needed for the viseme pipeline.
