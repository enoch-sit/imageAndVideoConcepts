EchoMimic — Audio-conditioned
Portrait Animation

EchoMimic (2024) animates a portrait using audio and optional facial landmark control. It extends AnimateDiff with an audio encoder branch and a landmark-conditioning adapter, enabling both audio-only and audio+landmark-guided animation modes.

Architecture — AnimateDiff + Audio

EchoMimic builds on AnimateDiff's temporal motion module architecture. It adds two adapters: an Audio Adapter that injects audio features via cross-attention, and a Facial Landmark Adapter (LandmarkCtrl) that optionally injects landmark sequences via ControlNet-style copying.

Step 1 of 5

Base: AnimateDiff Motion Modules

AnimateDiff inserts temporal self-attention modules between every spatial attention block of a U-Net. These 1D temporal attention layers operate on the T-frame axis, creating smooth frame-to-frame transitions without per-video fine-tuning. EchoMimic inherits this structure and adds conditioning adapters alongside it.

Audio Adapter — wav2vec2 Cross-Attention

Audio is encoded by a pre-trained wav2vec2 model (same as DAWN). EchoMimic's audio adapter projects these features and adds them via cross-attention at each temporal attention block. Only the adapter weights are trained; the AnimateDiff motion module weights are frozen during audio-specific training.

Audio adapter injection A = wav2vec2(audio_clip) ∈ ℝ^(T_a×768)
A_proj = Linear(A) ∈ ℝ^(T_a×C)
z'_t = TemporalAttn(z_t) + AudioCrossAttn(z_t, A_proj)

LandmarkCtrl — Optional Landmark Guidance

EchoMimic offers a second conditioning mode where facial landmarks (MediaPipe 478-point face mesh sampled to 68 key points) from a reference driving video are used as additional spatial guidance. This "landmark control" improves lip-sync accuracy at the cost of requiring a driving video.

ControlNet-style Landmark Adapter

The landmark adapter is a lightweight zero-initialized ControlNet. The landmark sequence (68 key points rendered as a heatmap image, one per frame) is encoded by the ControlNet encoder and added to the U-Net encoder features via zero-convolution residuals. This allows controlled disabling at inference for audio-only mode by zeroing the ControlNet weights.

LandmarkCtrl injection L_img_t = renderLandmarks(lm_t) — heatmap image
ctrl_feat = LandmarkEncoder(L_img_t)
z_enc_t += ZeroConv(ctrl_feat) — additive residual
disable = set ZeroConv weight → 0 for audio-only mode

Course usage of EchoMimic
EchoMimic is used in the course demo pipeline when a reference talking video is available (e.g. a known speaker video). The audio-only mode is used for production runs. The landmark mode is demonstrated as a teacher run to show how explicit face-mesh control improves lip accuracy, before introducing the full phoneme-to-viseme pipeline (Apps 24–25) which achieves similar control without a driving video.
