3D Rotary Position Embedding

Video tokens exist in three dimensions: time (T), height (H), width (W). 3D-RoPE extends the 1D RoPE from language models to encode all three simultaneously, so the transformer understands "frame 3, row 12, column 5" without separate learned position tables.

How 3D-RoPE Works

The Q and K vectors in each attention head are split into three equal segments. Each segment is rotated by a complex phase determined by one axis: time position, height position, or width position. Tokens at nearby (t,h,w) positions will have similar rotations and thus high dot products, encoding spatial and temporal proximity into the attention scores automatically.

3D-RoPE encoding (per head) Q = [RoPE_t(Q₁) ; RoPE_h(Q₂) ; RoPE_w(Q₃)]
K = [RoPE_t(K₁) ; RoPE_h(K₂) ; RoPE_w(K₃)]
attn(i,j) ∝ exp(Qᵢᵀ·Kⱼ/√d) encodes (Δt,Δh,Δw) via phase
Why 3D-RoPE outperforms learned position tables
Learned tables are fixed-size — a model trained on 49 frames cannot handle 97 frames. RoPE extrapolates naturally to longer sequences because it encodes relative positions via rotation angles, not lookup indices. CogVideoX can generate variable-length clips without retraining.

Expert Adaptive LayerNorm

Each transformer layer in CogVideoX uses Adaptive LayerNorm (AdaLN) conditioned on both the diffusion timestep t and the text embeddings. This allows the same layer weights to adapt their normalisation for different conditions.

Text Conditioning in CogVideoX

CogVideoX uses T5-XXL (same as FLUX.1) to encode text. The text tokens are prepended to the sequence of video latent tokens in the transformer — they share the same 3D-RoPE time dimension (assigned t=0, placed before the actual video frames). This lets every video token attend to text tokens naturally in the same attention operation.

Token sequence layout [T5_text_tokens | video_latent_tokens (T×H×W)]
T5 tokens: ~226 tokens at t=0 position
Video tokens: T/4×H/8×W/8 after temporal VAE

Course usage context
CogVideoX-2B is used in this course as the primary "talking head video" generation backbone. Given a driving audio and face region latent, the model generates a 49-frame clip of the face speaking. The temporal VAE and 3D transformer ensure smooth motion while T5-XXL ensures precise caption adherence for lipsync quality.
