DAWN — Diffusion for
Talking-Head Animation

DAWN (Diffusion-based Audio-driven talking head With motioN) removes the 3DMM stage. It directly conditions a latent diffusion model on audio embeddings and a single reference image, generating temporally consistent talking-head videos end-to-end.

Latent diffusion Audio conditioning No 3DMM required
01

Architecture Overview

DAWN uses a U-Net-based latent video diffusion model. A reference encoder extracts appearance features from the portrait image. An audio encoder converts audio mel features into conditioning vectors. Both are injected into the U-Net via cross-attention at every block.

Reference Image Encoder

The single portrait image is encoded by a frozen CLIP ViT-L/14 image encoder + a learnable adapter. This produces appearance tokens r ∈ ℝ^(256×C) that capture identity, hair, lighting, and background. These tokens are used in cross-attention at every U-Net decoder block, allowing the model to "copy" visual details from the reference into each generated frame.

Reference encoding r = ReferenceEncoder(I_ref) ∈ ℝ^(256×C)
cross_attn_out = Attention(Q=z_noisy, K=r, V=r)

Audio Encoder

DAWN uses a pre-trained wav2vec2 audio encoder (not DeepSpeech2 like SadTalker). wav2vec2 produces contextual speech representations at ~50 Hz. These are projected to the U-Net's hidden dimension and injected via temporal cross-attention — each denoising step attends to the corresponding audio time window.

Audio conditioning a_t = AudioEncoder(mel_t) ∈ ℝ^(T_audio×D_a)
a_t_proj = Linear(a_t) ∈ ℝ^(T_audio×C)
temporal_attn = Attention(Q=z_t, K=a_proj, V=a_proj)
02

Temporal Noise Injection for Motion

A key DAWN innovation is temporal noise injection: instead of diffusing pure Gaussian noise, DAWN adds structured temporal noise correlated across frames. This forces the model to learn smooth inter-frame transitions and prevents flickering without explicit optical flow or 3DMM constraints.

Temporal Noise Correlation
Adjust the frame correlation strength α. α=0 means independent noise per frame (flickery); α=1 means identical noise (no motion). DAWN uses α≈0.5 to balance temporal coherence and expressiveness.

Correlated Noise Construction

DAWN constructs a shared noise component ε_shared drawn once at the start for all frames, and individual independent noise ε_i per frame. The final noise for frame i is interpolated:

Temporal noise injection (DAWN) ε_shared ~ N(0, I) — drawn once for all T frames
ε_i ~ N(0, I) — independent per frame
ε̃_i = α·ε_shared + √(1-α²)·ε_i
— keeps same marginal distribution N(0,I)
— α controls frame-to-frame correlation

DAWN vs SadTalker Comparison

Both systems animate a portrait from audio, but they take fundamentally different paths. SadTalker uses an explicit 3D intermediate representation; DAWN is fully end-to-end in pixel/latent space with no 3D geometry.

When to use DAWN vs SadTalker
SadTalker: better identity preservation for lip-sync accuracy; the 3DMM stage explicitly models jaw/lip geometry so phoneme→viseme control is more precise. DAWN: more photo-realistic background and hair consistency since it stays in pixel/latent space; better for natural head motion diversity and generating from arbitrary-angle portraits. In the course pipeline, SadTalker is used for production lip-sync (App 27) because of better phoneme control.
