01

Stochastic → Deterministic

DDIM sampling converts the stochastic reverse diffusion process into a deterministic probability flow ODE, making generation reproducible from the same latent seed.

The DDPM Reverse Process

DDPM (Denoising Diffusion Probabilistic Models) generates images by reversing a noise-adding process. Starting from pure Gaussian noise $x_T$, it removes a small amount of noise at each timestep. Each step is a stochastic Markov chain:

DDPM reverse step $$x_{t-1} = \frac{1}{\sqrt{\alpha_t}} \left( x_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}} \,\epsilon_\theta(x_t,t) \right) + \sigma_t \, z$$

Here $\epsilon_\theta(x_t,t)$ is the U-Net's predicted noise, and $z \sim \mathcal{N}(0,I)$ is fresh random noise injected at every step. The $\sigma_t z$ term is what makes DDPM stochastic — running the same seed twice yields different images.

The DDIM Reformulation

Song et al. (2020) made a key observation: if we set $\sigma_t = 0$, we can reformulate the reverse process as a deterministic update. First, predict the clean image $\hat{x}_0$ at each step:

Clean image prediction $$\hat{x}_0 = \frac{x_t - \sqrt{1-\bar{\alpha}_t} \,\epsilon_\theta(x_t,t)}{\sqrt{\bar{\alpha}_t}}$$

Then the deterministic update becomes:

DDIM deterministic step $$x_{t-1} = \sqrt{\bar{\alpha}_{t-1}} \,\hat{x}_0 + \sqrt{1-\bar{\alpha}_{t-1}} \,\epsilon_\theta(x_t,t)$$

This is exactly the Euler discretization of the probability-flow ODE:

Probability-flow ODE $$\frac{dx}{dt} = \mathbf{v}(x_t, t)$$

No stochastic term. The entire trajectory is a smooth, deterministic flow through latent space — a single vector field guides $x_T$ all the way to $x_0$.

Reproducibility Consequence

Because there is no injected noise, the mapping from latent code to output is fully deterministic:

Key Insight
Same latent seed $x_T \sim \mathcal{N}(0,I)$ + same model $\epsilon_\theta$ → identical ODE trajectory → pixel-identical output every run.

This enables seed-based workflows: you can share a seed to reproduce an exact image, interpolate between seeds to morph between outputs, or build deterministic editing pipelines — none of which are possible with stochastic DDPM.

02

1 000 Steps → 20 Steps

The ODE formulation reduces sampling from 1 000 stochastic steps to just 20–50 deterministic steps with no measurable quality loss.

Why DDPM Needs ~1 000 Steps

Each noise injection $\sigma_t z$ accumulates variance. When you try to take large steps (fewer total steps), the accumulated noise pushes the sample far off the data manifold.

  • Large $\Delta t$ → large noise per step → the sample drifts off-course
  • Quality metrics (FID, CLIP score) collapse rapidly below ~500 steps
  • DDPM is fundamentally limited by this variance-accumulation problem

The ODE Advantage

With no injected variance, the ODE path is the mean trajectory of the original SDE. The U-Net learns a smooth vector field $\mathbf{v}(x_t,t)$ that remains accurate even with coarse timesteps.

Mathematical Justification
The probability-flow ODE has the same marginal distributions $p_t(x)$ as the original SDE, but follows a single deterministic trajectory instead of averaging over noise. You can skip large chunks of the path and still land accurately.

Empirical Discretization & Modern Schedulers

DDIM uses non-Markovian sampling with arbitrarily large $\Delta t$ (e.g., stride = 20–50). Modern schedulers push this further:

  • DPM++ 2M Karras — second-order multi-step solver
  • Euler a — adaptive Euler with ancestral sampling
  • Quality stays within 0.1–0.5 FID of full 1 000-step DDPM

Fast Enough for Everyone

Following a smooth ODE path — instead of adding random noise at every step — is what makes Stable Diffusion fast enough to run on consumer hardware.

The Computational Bottleneck

Each diffusion step requires one full U-Net forward pass, which constitutes ~90% of total inference runtime. The number of steps directly controls how long generation takes.

Step Reduction Impact

DDPM (1 000 steps)
30–60 s
on RTX 4090 (512×512)
DDIM / ODE (20–50 steps)
1–3 s
20–50× wall-clock speedup

Latent Space Synergy

Stable Diffusion compounds the speedup by running diffusion in a compressed latent space. A VAE encoder maps $512 \times 512 \times 3$ pixel images to $64 \times 64 \times 4$ latents — an 8× spatial compression:

512 × 512 × 3
→
64 × 64 × 4

Combined with ODE sampling, total inference drops to ~0.5–2 s on mid-range GPUs and Apple Silicon.

Why This Matters
  • No random noise → aggressive step schedules are stable (no variance explosion)
  • Trajectory stays coherent across large $\Delta t$ jumps → fewer U-Net evaluations needed
  • Enables real-time consumer deployment: DDPM was research-lab only; DDIM + latent space gave us Stable Diffusion — the first practical consumer text-to-image model
