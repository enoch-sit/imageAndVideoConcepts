The three main blocks run in sequence: encode the text prompt → denoise latent → decode to pixels.

Why Latent Space?

SD 1.5's key innovation over DDPM is running the diffusion in latent space. Instead of denoising 786,432 pixel values, the U-Net works on 16,384 latent values — a 48× dimensionality reduction. This makes 50-step generation feasible in under 10 seconds on a consumer GPU.

Compute comparison DDPM pixel diffusion: 1000 × U-Net(786k values)
SD 1.5 latent diffusion: 50 × U-Net(16k values) + decode once

CLIP Conditioning via Cross-Attention

The text embedding from CLIP is injected into every attention block of the U-Net via cross-attention. The noisy latent features serve as the query (Q), while the text embedding provides the keys (K) and values (V). This mechanism lets the U-Net attend differently to different words in the prompt at each spatial location.

Cross-attention (per U-Net block) Q = W_Q · z_spatial (from noisy latent)
K = W_K · c_text (from CLIP embedding)
V = W_V · c_text
out = softmax(QK^T / √d) · V
03

CLIP Text Encoder

CLIP (Contrastive Language-Image Pre-training) maps text into a 768-dimensional space pre-aligned with image features — giving the U-Net a rich semantic anchor.

Tokenisation and Context Length

SD 1.5 uses the CLIP ViT-L/14 text encoder. The prompt is tokenised to at most 77 tokens (including start and end). Each token maps to a 768-dimensional vector. The entire 77×768 matrix is the condition embedding c passed to the U-Net.

Prompt → Condition Tensor prompt → [77 tokens] → CLIP_L → c ∈ R^{77 × 768}

Context Limit: 77 Tokens

Prompts longer than 77 tokens are silently truncated by CLIP. Each token is roughly one word or sub-word. "a photograph of a beautiful sunset over the mountains with warm golden light" is about 14 tokens — well under the 75-token limit (77 minus start/end). SDXL resolves this by using two separate encoders with a 154-token combined budget.

Practical tokenisation check
Run len(tokenizer.encode(prompt)) before generation. Anything above 77 will be clipped. Use natural, comma-separated concrete nouns and adjectives — CLIP was trained on image–caption pairs so those semantic patterns transfer best.

Key Numbers at a Glance

  • U-Net: 860M parameters, 4-channel latent input
  • CLIP text encoder: 123M parameters, frozen during SD training
  • VAE: 83M parameters (encoder + decoder)
  • Default output: 512×512 px (can be pushed to 768 with minor quality drop)
  • A16 GPU (16 GB VRAM): ~5 sec/image at 20 DDIM steps
